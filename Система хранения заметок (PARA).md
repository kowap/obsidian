**PARA** — это популярная система организации заметок, файлов и информации, созданная экспертом по продуктивности **Тьяго Форте (Tiago Forte)**. Она используется в системах вроде **Notion, Obsidian, Apple Notes, Evernote** и является частью концепции **“Second Brain” (второй мозг)**

Главная идея:
> хранить информацию **не по типу (статьи, идеи, заметки)**, а по **степени её полезности для текущих задач**.

Название **PARA** — это аббревиатура из четырёх категорий.

# **Структура PARA**

## **1. Projects (Проекты)**

**Краткосрочные задачи с конкретной целью.**  

Проект — это то, что:
- имеет **четкий результат**
- имеет **конец**
- требует действий.

Примеры:
- Сделать сайт
- Подготовить презентацию
- Записать курс
- Сделать ремонт

## **2. Areas (Области ответственности)**

Это **долгосрочные области жизни**, за которые ты постоянно отвечаешь.
Они **не имеют конца**, но требуют поддержания.

Примеры:
- Финансы
- Здоровье
- Работа
- Семья
- Обучение

## **3. Resources (Ресурсы)**

Это **информация и знания**, которые могут быть полезны.

Это **не задачи**, а **справочный материал**.

Примеры:
- заметки по Laravel
- статьи о SEO
- идеи проектов
- исследования
- полезные ссылки

## **4. Archives (Архив)**

Сюда отправляется всё **неактивное**.

Например:
- завершённые проекты
- старые ресурсы
- закрытые темы

```
Archive
 ├── Old projects
 ├── Past ideas
 └── Old research
```

Архив нужен, чтобы:
- не удалять информацию
- не загромождать активные разделы.

# **Как работает система (логика)**

Любая информация попадает в **одну из 4 категорий**.

Например:

| **Информация**       | **Куда** |
| -------------------- | -------- |
| План запуска сайта   | Project  |
| Документация Laravel | Resource |
| Финансовый учёт      | Area     |
| Завершённый проект   | Archive  |

Главное правило:

> **Проекты — главный центр системы.** 

Все ресурсы и заметки должны помогать **конкретным проектам**.

# **Почему PARA популярна**

Плюсы:

**1. Очень простая структура**
Всего 4 папки.

  

**2. Работает в любом приложении**
- Notion
- Obsidian
- Apple Notes
- Google Drive
- файловая система


**3. Фокус на действиях**
Информация хранится не ради хранения, а ради **использования**.

**4. Меньше хаоса**
Ты всегда знаешь:
- куда положить заметку
- где её искать.


Example

```
Projects
 ├── Domain CRM
 ├── Static Site Generator
 └── Poker Calculator App

Areas
 ├── Programming
 ├── Health
 └── Personal finance

Resources
 ├── Laravel
 ├── AI prompts
 ├── SEO
 └── DevOps

Archive
 ├── Old generator v1
 └── Closed projects
```
